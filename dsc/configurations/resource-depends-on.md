# <a name="resource-dependencies-using-dependson"></a>Závislosti prostředků pomocí DependsOn

Při psaní [konfigurace](configurations.md), přidáte [prostředků bloky](../resources/resources.md) pro konfiguraci aspektů cílový uzel. Jak vám pokračujte přidáním prostředků bloky, vaše konfigurace můžou růst poměrně velké a těžkopádný proces pro správu. Jeden takový problém je použité pořadí bloků prostředků. Obvykle prostředky se použijí v pořadí, ve kterém jsou definovány v rámci konfigurace. S růstem vaší konfigurace větší a složitější, můžete použít `DependsOn` klíč změníte použitých prostředků tak, že zadáte prostředku, na kterých závisí jiný prostředek.

`DependsOn` Klíč lze použít v jakékoli prostředků bloku. Je definován pomocí mechanismu klíč/hodnota jako další klíče prostředku. `DependsOn` Klíč očekává, že pole řetězců s následující syntaxí.

```
DependsOn = '[<Resource Type>]<Resoure Name>', '[<Resource Type>]<Resource Name'
```

Následující příklad nakonfiguruje pravidlo brány firewall po povolení a konfigurace veřejný profil.

```powershell
# Install the NetworkingDSC module to configure firewall rules and profiles.
Install-Module -Name NetworkingDSC

Configuration ConfigureFirewall
{
    Import-DSCResource -Name Firewall, FirewallProfile
    Node localhost
    {
        Firewall Firewall
        {
            Name                  = 'IIS-WebServerRole-HTTP-In-TCP'
            Ensure                = 'Present'
            Enabled               = 'True'
            DependsOn             = '[FirewallProfile]FirewallProfilePublic'
        }

        FirewallProfile FirewallProfilePublic
        {
            Name = 'Public'
            Enabled = 'True'
            DefaultInboundAction = 'Block'
            DefaultOutboundAction = 'Allow'
            AllowInboundRules = 'True'
            AllowLocalFirewallRules = 'False'
            AllowLocalIPsecRules = 'False'
            NotifyOnListen = 'True'
            LogFileName = '%systemroot%\system32\LogFiles\Firewall\pfirewall.log'
            LogMaxSizeKilobytes = 16384
            LogAllowed = 'False'
            LogBlocked = 'True'
            LogIgnored = 'NotConfigured'
        }
    }
}

ConfigureFirewall -OutputPath C:\Temp\
```

V pokročilejší konfigurace, můžete také použít [napříč závislost uzlu](crossNodeDependencies.md) umožňující ještě podrobnější řízení (například zajištění řadič domény je nakonfigurován než klienta připojíte k doméně).

## <a name="cleaning-up"></a>Čištění

Pokud jste použili výše uvedené konfiguraci, můžete se obrátit klíče vrátit zpět všechny změny. V příkladu výše, nastavení **povoleno** klíče na hodnotu false zakáže pravidlo brány firewall a profilu. V příkladu by měl změnit podle potřeby tak, aby odpovídaly předchozí nakonfigurovaný stav cílový uzel.

```powershell
        Firewall Firewall
        {
            Name                  = 'IIS-WebServerRole-HTTP-In-TCP'
            Enabled               = 'False'
            DependsOn             = '[FirewallProfile]FirewallProfilePublic'
        }

        FirewallProfile FirewallProfilePublic
        {
            Name = 'Public'
            Enabled = 'False'
        }
```
